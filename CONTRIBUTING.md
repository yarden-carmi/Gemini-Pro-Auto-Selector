# Contributing to Gemini Pro Auto-Selector

Thanks for your interest in contributing to Gemini Pro Auto-Selector! We welcome all types of contributions, including bug reports, feature suggestions, and code improvements.

## Reporting Bugs

Before reporting a bug, please search existing issues to see if it has already been reported. When reporting a bug, please include:
- A clear and descriptive title.
- Steps to reproduce the issue.
- Your browser and version.
- Any relevant logs or screenshots.

## Feature Requests

We're always looking for ways to improve the extension. If you have an idea for a new feature, please open an issue and describe:
- The problem you're trying to solve.
- Your proposed solution.
- Any alternatives you've considered.

## Code Contributions

1. Fork the repository.
2. Create a new branch for your feature or bug fix.
3. Commit your changes.
4. Push your branch to your fork.
5. Create a pull request.

Please ensure your code follows the existing style and is well-documented.

## Development Setup

To test your changes locally, follow the development installation instructions in the [README.md](README.md).
